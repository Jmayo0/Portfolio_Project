Drinks API

On Slide 1, users can interact with a feature that helps them find information about different drinks. Here's how it works:

  • Search Bar Interaction: Users can type the name of a drink they're curious about into a search bar.
  • Display Area: After typing, the code starts searching for the drink's details. It then displays this information in a special area on the webpage.
  • Drink Details: For each drink found, users can see details like the name, category, alcoholic content, and how it's served. The code goes through a list of ingredients needed for the drink and shows them too. It also provides instructions on how to make the drink.
  • Drink Image: Along with the details, users can see a picture of the drink next to the information.
  • Feedback for No Results: If the drink name doesn't match anything in the search, the code lets users know that it couldn't find anything. It also hides the picture so the page looks neat.

Behind the scenes, the code, written in JavaScript, looks at the user's input, searches through a big list of drinks using an external API called "The Cocktail DB," and organises the information neatly for display. It's like having a personal drink expert right on the webpage!

Snake Game

On Slide 2, users engage with an interactive snake game. Here's a breakdown of what they experience:

  • Game Initialisation: When the slide loads, the code sets up the environment for the snake game. It prepares the game canvas and initialises the game components.
  • Snake Movement: Users control the snake's movement using arrow keys on their keyboard. The snake moves around the game area, eating food and growing longer as it does so.
  • Game Mechanics: The snake can wrap around the edges of the game area, reappearing on the opposite side if it goes off-screen. When the snake eats food, it grows longer, and the player's score increases.
  • Game Pausing: Users can pause and resume the game by clicking the "Pause" button displayed on the screen. This button toggles between "Pause" and "Play" depending on the game's current state.
  • Visual Feedback: The game provides visual feedback through console logs, indicating actions such as snake movement, food placement, and game initialisation.

Behind the Scenes: The game is built using JavaScript and utilises the Three.js library for rendering 3D graphics. It employs concepts like scene setup, camera positioning, and object manipulation to create an immersive gaming experience.

Music Visualisation

On Slide 3, users are treated to a mesmerising music visualisation experience. Here's what they'll encounter:

  • Three.js Scene Setup: The slide initialises a Three.js scene, setting up a 3D environment for the visualisation.
  • Visual Elements: A large sphere and a textured environment are created to serve as the primary visual elements.
  • Lighting: Lighting is added to the scene to enhance the visual quality of the elements.
  • Audio Integration: Users can play music by interacting with the "Play" button. The music is synchronised with the animation for a cohesive experience.
  • Dynamic Animation: The sphere rotates and scales dynamically in response to the music, creating visually stunning effects.

Behind the Scenes: The visualisation leverages Three.js and HTML5 audio capabilities to create an immersive experience. It combines 3D rendering techniques with audio analysis to synchronise the animation with the music, resulting in a captivating display.

Premier League Stadium Map

On Slide 4, users can explore an interactive map showcasing Premier League stadiums. Here's what they'll find:

  • Map Interaction: Users can navigate the map to view the locations of various Premier League stadiums.
  • Marker Display: Each stadium is marked on the map with a pin icon for easy identification.
  • Info Window: Clicking on a pin opens an info window displaying details about the club and stadium.
  • Data Source: The stadium data is fetched from a CSV file hosted online, ensuring the information is up-to-date.

Behind the Scenes: The map is powered by the Google Maps JavaScript API, allowing seamless integration of interactive features. JavaScript fetches stadium data from a CSV file, parses it, and dynamically creates markers and info windows for each stadium location.

Matrix Calculator

On Slide 5, users can access a matrix calculator allowing them to perform various matrix operations. Here's what they'll find:

  • Matrix Input: Users can input matrices by entering values separated by commas and rows separated by new lines.
  • Operations: Users can perform operations such as transpose, determinant calculation, addition, subtraction, and multiplication on the matrices.
  • Custom Modal: When performing binary operations (addition, subtraction, multiplication), a modal prompts users to input the second matrix.
  • Result Display: The result of the matrix operation is displayed below the input area in a formatted manner.

Behind the Scenes: The calculator is implemented using JavaScript, parsing user input to perform matrix operations. When performing binary operations, a custom modal is shown to facilitate input of the second matrix. The result is then displayed dynamically on the webpage.
